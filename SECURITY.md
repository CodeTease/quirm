# Security Policy

## Supported Versions

We are committed to keeping our users safe. Below are the versions of Quirm that currently receive security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.5.x+   | ✅ Supported       |
| 0.4.x   | ⚠️ Critical Only   |
| < 0.3.x | ❌ Not Supported   |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability within this project, please send an email to **contact@teaserverse.online**. 

To help us resolve the issue as quickly as possible, please include:
* A description of the vulnerability.
* Steps to reproduce the issue (proof of concept).
* Potential impact of the vulnerability.

We aim to acknowledge your report within **1 week** and will keep you updated on our progress as we work to resolve the issue.

## Our Security Process

1. **Acknowledgment:** We will confirm receipt of your report.
2. **Investigation:** Our team will investigate the issue and determine its severity.
3. **Fix:** We will work on a patch to address the vulnerability.
4. **Release:** We will release a new version containing the fix.
5. **Announcement:** Once the fix is public, we will give credit to the reporter (if desired).

Thank you for helping keep this project secure for everyone! ❤️
